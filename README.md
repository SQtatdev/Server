# Server
---
  1. normal (single-threaded) server
üìú How it works:
- The server handles clients one at a time.  
- While one client interacts with the server, the others wait.  

‚úÖ Pros:  

‚úî Simple code, easy to understand and debug.  
‚úî Minimal resource utilization.  

‚ùå Minuses: 
‚úñ Only one client at a time.  
‚úñ If a client takes a long time to respond, the server hangs.  

When to use.  
üîπ For testing or if clients rarely connect.  

---

  2. Multi-threaded server (`threading`)
üìú How it works:  
- A separate thread is created for each client.  
- The server can handle multiple clients at the same time.  

‚úÖ Pros:  

‚úî Simple implementation.  
‚úî Supports multiple clients at the same time.  

‚ùå Minuses:  
‚úñ Heavy load on the system because each thread requires memory.  
‚úñ The system may **slow down** if there are a large number of clients.  

When to use.  
üîπ If there are few connections (up to **1000 clients**).  

---

  3. asynchronous server (`asyncio`)
üìú How it works:  
- Uses asynchronous operations (`await`) without creating threads.  
- The server processes all clients **simultaneously** without blocking.  

‚úÖ Pros:   

‚úî Handles thousands of connections efficiently.  
‚úî Uses fewer resources than streams.  

‚ùå Minuses: 
‚úñ More difficult to understand and debug.  
‚úñ Requires Python 3.7+.  

When to use it?  
üîπ For busy servers and a large number of clients.
